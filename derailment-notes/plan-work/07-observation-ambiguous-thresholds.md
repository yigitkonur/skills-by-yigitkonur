# Observation: Ambiguous Thresholds

Date: 2025-07-12
Related friction points: F-01, F-08, F-11, F-25, F-26

---

The skill uses qualitative thresholds where concrete criteria would be more actionable:
- "Pick one dominant job" -- but what if two are equally dominant?
- "success cannot be observed" -- what counts as observable?
- "closes a clear gap" -- how clear is clear?
- "enough evidence" -- how much is enough?
- Boundary with run-research -- when does plan-work end and run-research begin?

## Fixes Applied

- F-01: Added precedence ordering (diagnose > compare > forecast > frame > align > explore) and tiebreaker ("when tied, choose the job whose output the next decision-maker needs first")
- F-08: Added "(no metric, test, or outcome you could check)"
- F-11: Replaced with "cannot answer a question the user explicitly asked"
- F-25: Added "can fill every cell of method template + articulate why leader wins" heuristic
- F-26: Added explicit context boundary: "plan-work = immediate context and structured reasoning; run-research = needs external data, benchmarks, or literature"
