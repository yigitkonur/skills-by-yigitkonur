# Planning Mission Template

Use this when the deliverable is a plan, not code changes.

```markdown
## Context

What needs to be built or fixed, why it matters, and what already exists.

## Mission

Produce an implementation plan only. Do not modify files.

## Constraints

- Each task should have a clear file scope
- Dependencies between tasks must be explicit
- The plan should separate parallel work from sequential work

## Required Output

### Overview

One short paragraph describing the plan.

### Wave 1

Task 1:
- Files:
- Goal:
- Definition of done:

### Wave 2

Task 2:
- Files:
- Goal:
- Definition of done:

### Verification

Which commands should be run after implementation.
```
