# Makefile base — universal preamble, ANSI palette, helper conventions

Every Makefile generated by this skill begins with the same preamble, the same ANSI palette, the same `make help` pattern, and the same helper-naming convention. This file is the canonical source for all of it. Scenario-specific recipes (`local`, `tunnel`, `deploy*`, etc.) live in their own references; this one covers the parts shared across all seven scenarios.

## Universal preamble

Paste this at the top of every generated Makefile. The lines are not optional — each one prevents a specific failure mode.

```makefile
SHELL       := bash
.SHELLFLAGS := -eu -o pipefail -c
.ONESHELL:
.DELETE_ON_ERROR:
MAKEFLAGS   += --warn-undefined-variables --no-builtin-rules
.DEFAULT_GOAL := help
```

| Line | Why | Failure mode it prevents |
|---|---|---|
| `SHELL := bash` | Forces bash, not the system `/bin/sh` (often `dash` on Debian/Ubuntu, which lacks `[[`, arrays, `pipefail`). Recipes that work on macOS would silently fail on Ubuntu CI without this. | `[[ -z "$X" ]]` syntax error in `sh`; arithmetic `$(( ))` quirks; `pipefail` not honoured. |
| `.SHELLFLAGS := -eu -o pipefail -c` | `-e` exit on first error; `-u` unset-variable is an error; `-o pipefail` propagates pipe failures. The `-c` is required because Make passes the recipe as a single string. | Without `-e`: a failing `cd` followed by `rm -rf` deletes the wrong dir. Without `-o pipefail`: `false \| cat` returns 0 — a failed deploy silently looks green. |
| `.ONESHELL:` | Whole recipe runs in ONE shell invocation. Variables (`$$pid=...`) survive across lines without `\` continuations. | Without it, every line is a fresh shell — `pid=...; kill $$pid` becomes two recipes; the variable is lost. |
| `.DELETE_ON_ERROR:` | If a recipe fails midway, Make deletes the partial target. | A half-written `dist/bundle.js` would otherwise be treated as up-to-date on the next run, masking the build failure. |
| `MAKEFLAGS += --warn-undefined-variables` | Surfaces typos in variable names. Make would otherwise expand `$(PROOT)` (typo of `$(PROJECT)`) to empty silently. | `rm -rf $(PROOT)/build` becoming `rm -rf /build` on a typo. |
| `MAKEFLAGS += --no-builtin-rules` | Disables Make's implicit rules (`%.o: %.c`, etc.). They never apply to deploy/dev workflows and just slow startup + create surprising matches. | `make foo.bar` accidentally invoking a builtin pattern. |
| `.DEFAULT_GOAL := help` | A bare `make` (no target) prints help instead of running the first target by accident. | A user typing `make` and triggering a deploy or wipe by mistake. |

Source: Davis-Hansson "Your Makefiles are wrong" (`tech.davis-hansson.com/p/make/`); GNU Make manual on `.ONESHELL`, `.DELETE_ON_ERROR`, `MAKEFLAGS`. The pragmatic version (no `.RECIPEPREFIX`) is the right default — keeps tab-indented recipes that any editor can handle.

## ANSI palette

```makefile
# ANSI palette (no-op on dumb terminals — esc sequences are ignored when stdout is not a TTY)
B := \033[1m
D := \033[2m
G := \033[32m
Y := \033[33m
R := \033[31m
C := \033[36m
Z := \033[0m
```

| Code | Meaning | Use for |
|---|---|---|
| `B` | bold | section titles, target names in `make help` |
| `D` | dim | hints, low-priority detail, "stop:" footer lines |
| `G` | green | success messages, primary command in help |
| `Y` | yellow | warnings, "stale dev — reclaiming" |
| `R` | red | refusals, errors, missing-token gate |
| `C` | cyan | URLs, secondary commands |
| `Z` | reset | always close every coloured run with `$(Z)` |

The escape sequences are literal `\033[…m` strings. Print with `printf` (not `echo` — `echo -e` is non-portable). Dumb terminals and CI logs strip the codes and you see the underlying text — no garbled output.

## `make help` — self-documenting recipe

Every public target has a trailing `## description` comment. The `help` target greps the Makefile for those markers and aligns them. Pattern (combining Davis-Hansson `tech.davis-hansson.com/p/make/` and `gist.github.com/prwhite/8168133`):

```makefile
help: ## show this help and exit
	@printf "$(B)$(PROJECT) $(Z)$(D)— make targets$(Z)\n\n"
	@grep -E '^[a-zA-Z_-]+:.*?## .*$$' $(MAKEFILE_LIST) \
		| awk 'BEGIN{FS=":.*?## "} {printf "  $(C)%-18s$(Z) %s\n", $$1, $$2}'
	@printf "\n$(D)Override knobs:  PORT=4000  TUNNEL_PORT=4001  HOST=0.0.0.0$(Z)\n"
```

Rules:
- `## ` (two hashes + space) is the marker. Targets without it don't appear.
- The regex `^[a-zA-Z_-]+:` matches public targets only — helpers prefixed with `_` are excluded.
- The `awk` separator `:.*?## ` peels the description out cleanly even when the recipe has prerequisites (`local: _free-port-$(PORT) ## start dev server`).
- `$(MAKEFILE_LIST)` includes any included sub-Makefiles, so a monorepo root with `include apps/web/Makefile` documents itself + the included one.

Example public targets with the marker:

```makefile
local: _free-port-$(PORT) _ensure-deps ## dev server on http://localhost:$(PORT)
local-lan: _free-port-$(PORT) ## dev server on $(HOST):$(PORT) for phone-on-Wi-Fi
tunnel: ## tailscale serve → http://<node>.ts.net (tailnet only)
stop: ## kill our own dev process on $(PORT)
clean: stop ## stop + wipe build/cache dirs
deploy: deploy-vercel ## production deploy
verify: ## HTTP-probe the deployed URL
```

## `.PHONY` discipline

Every non-file target must be listed in `.PHONY`. Make's default rule is "run the recipe only if the target is older than its prerequisites" — but `local`, `deploy`, `clean` aren't files, so Make has nothing to compare and gets confused.

```makefile
.PHONY: help local local-lan tunnel tunnel-stop stop clean \
        deploy deploy-vercel deploy-railway verify env-pull \
        _free-port-% _ensure-deps _check-vercel-tokens _print-banner-local
```

Concrete failure mode: without `.PHONY: build`, if the project has a `build/` directory, `make build` would silently no-op because Make sees the directory and assumes it's up-to-date. The user retries, retries, swears, and the build never runs.

Rules:
- Add a target to `.PHONY` the moment you write it. Don't batch up; you'll forget one.
- Helpers (`_free-port-%`) are still phony. Pattern rules need `.PHONY` too.
- `.PHONY: %` is tempting but wrong — it disables file-based incremental rules globally.

## Tabs vs spaces

Recipe lines MUST start with a literal TAB character. Spaces produce `*** missing separator. Stop.`

```makefile
target:
<TAB>command          # correct
target:
    command            # WRONG: 4 spaces → "missing separator"
```

Make 4.0+ supports `.RECIPEPREFIX := >` to override:

```makefile
.RECIPEPREFIX := >
target:
> command              # legal under .RECIPEPREFIX
```

This skill keeps tabs as the default for compatibility — macOS ships GNU Make 3.81 by default (no `.RECIPEPREFIX`), and most CI runners default to whatever `make` is on their image. Tabs always work.

If your editor strips tabs, configure it to keep tabs in `Makefile`. VSCode: add `"[makefile]": { "editor.insertSpaces": false, "editor.detectIndentation": false }` to settings. Vim: `:set noexpandtab`.

## Helper-target naming convention

| Prefix | Meaning | Listed in `make help`? |
|---|---|---|
| public name (`local`, `deploy`) | user-invokable | yes (with `## description`) |
| `_` prefix (`_free-port-%`, `_check-vercel-tokens`, `_print-banner-local`) | internal helper, only invoked by other recipes | no — the help-grep regex excludes them |

Helpers are still `.PHONY`. They still use the universal preamble. They just don't show up in `make help` because users don't call them directly. Examples:

```makefile
_free-port-%: # %=PORT — free a port if held by our own dev process
	# (full body in port-hygiene.md)

_check-vercel-tokens:
	@: "$${VERCEL_TOKEN:?set VERCEL_TOKEN before running deploy-vercel}"
	@: "$${VERCEL_PROJECT:?set VERCEL_PROJECT or run vercel link first}"

_print-banner-local:
	@printf "\n$(B)$(G)$(PROJECT) dev$(Z)  $(D)localhost only$(Z)\n"
	@printf "  $(C)→$(Z) http://localhost:$(PORT)\n"
	@printf "  $(D)stop:  make stop      cache panic:  make clean$(Z)\n\n"
```

## Default knobs

```makefile
PROJECT     ?= myapp           # set per-project; the help banner uses it
PORT        ?= 3456            # local dev port (avoid 3000 — OrbStack squat zone)
HOST        ?= 127.0.0.1       # localhost-only by default; LAN binding is opt-in
TUNNEL_PORT ?= 3457            # different from PORT so tunnel + local-lan can coexist
TUNNEL_TLS  ?= 0               # 1 = https=443, 0 = http=80
```

`?=` makes each one overridable per-invocation: `make local PORT=4000`. Defaults must work — see `references/scenario-detection.md` for the "no required arguments" rule.

## Minimal example

A complete, copy-paste-ready Makefile using only the universal targets. Drop in scenario-specific targets and helpers (from `makefile-frontend.md`, `makefile-backend.md`, etc.) on top.

```makefile
SHELL       := bash
.SHELLFLAGS := -eu -o pipefail -c
.ONESHELL:
.DELETE_ON_ERROR:
MAKEFLAGS   += --warn-undefined-variables --no-builtin-rules
.DEFAULT_GOAL := help

PROJECT     ?= myapp
PORT        ?= 3456
HOST        ?= 127.0.0.1

# ANSI palette
B := \033[1m
D := \033[2m
G := \033[32m
Y := \033[33m
R := \033[31m
C := \033[36m
Z := \033[0m

.PHONY: help local stop clean _free-port-% _print-banner-local

help: ## show this help and exit
	@printf "$(B)$(PROJECT) $(Z)$(D)— make targets$(Z)\n\n"
	@grep -E '^[a-zA-Z_-]+:.*?## .*$$' $(MAKEFILE_LIST) \
		| awk 'BEGIN{FS=":.*?## "} {printf "  $(C)%-18s$(Z) %s\n", $$1, $$2}'
	@printf "\n$(D)Override knobs:  PORT=4000  HOST=0.0.0.0$(Z)\n"

local: _free-port-$(PORT) _print-banner-local ## dev server on http://localhost:$(PORT)
	@HOST=$(HOST) PORT=$(PORT) npm run dev

stop: ## kill our own dev process on $(PORT)
	@pid=$$(lsof -ti :$(PORT) 2>/dev/null || true); \
	if [ -n "$$pid" ]; then \
	  kill -TERM $$pid 2>/dev/null || true; \
	  sleep 0.5; \
	  if kill -0 $$pid 2>/dev/null; then kill -9 $$pid 2>/dev/null || true; fi; \
	  printf "$(Y)stopped pid $$pid on :$(PORT)$(Z)\n"; \
	else \
	  printf "$(D)nothing on :$(PORT)$(Z)\n"; \
	fi

clean: stop ## stop + wipe build/cache dirs
	@rm -rf .next/cache dist build && printf "$(G)cleaned build/cache$(Z)\n"

_free-port-%:
	# body in port-hygiene.md

_print-banner-local:
	@printf "\n$(B)$(G)$(PROJECT) dev$(Z)  $(D)localhost only$(Z)\n"
	@printf "  $(C)→$(Z) http://localhost:$(PORT)\n"
	@printf "  $(D)stop:  make stop      cache panic:  make clean$(Z)\n\n"
```

This Makefile is functional on its own — `make` shows help; `make local` starts the dev server; `make stop` kills it; `make clean` wipes caches. Add the scenario-specific targets on top.
