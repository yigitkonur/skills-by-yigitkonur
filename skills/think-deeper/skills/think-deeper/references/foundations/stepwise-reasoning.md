# Stepwise Reasoning

Use numbered, testable steps instead of leaps.

## Core Rule

At each step, answer:
1. What do I know now?
2. What follows from that?
3. What is still assumed?
4. What is the next best step?

Then take only that step.

## Good Stepwise Reasoning

- each step depends on explicit evidence
- each step changes certainty, scope, or direction
- each step can be revised when new evidence arrives

## Bad Stepwise Reasoning

- skipping from symptom to cause
- naming a fix before naming the mechanism
- bundling multiple unresolved assumptions into one conclusion
- continuing after the next move is already obvious

## Practical Pattern

Use this loop:

1. observe
2. interpret cautiously
3. test interpretation
4. update the model
5. continue or stop

The point is not to create more steps. The point is to prevent invisible jumps.
